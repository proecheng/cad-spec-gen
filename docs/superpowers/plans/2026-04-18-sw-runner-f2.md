### Task 2.2：T2 — 轮询 runner 上线（C3 检查点）

**Files:** 无

- [ ] **Step 1：轮询 GitHub API**

```powershell
$max = 12
$transitionDetected = $false
for ($i = 1; $i -le $max; $i++) {
  try {
    $apiOut = gh api repos/proecheng/cad-spec-gen/actions/runners
    if (-not $apiOut) {
      Write-Warning "[$i/$max] gh api empty output, retry"
      Start-Sleep -Seconds 15; continue
    }
    $r = $apiOut | ConvertFrom-Json |
         Select-Object -ExpandProperty runners |
         Where-Object { $_.name -eq 'procheng-sw-smoke' }
    Write-Host "[$i/$max] status=$($r.status) busy=$($r.busy) (baseline was: $($baselineRunner.status))"
    if ($r.status -eq 'online' -and $baselineRunner.status -eq 'offline') {
      Write-Host "C3 PASS: runner transitioned offline→online"
      $transitionDetected = $true
      break
    }
  } catch {
    Write-Warning "[$i/$max] gh api error: $_"
  }
  Start-Sleep -Seconds 15
}

if (-not $transitionDetected) {
  Write-Warning "C3 超时：3 min 内未检测到 offline→online transition"
}
```

- [ ] **Step 2：判定 C3**

- ✅ **C3 PASS**：transition detected（输出 "C3 PASS: runner transitioned offline→online"）→ 继续 Task 2.3
- ❌ **C3 FAIL**：3 min 超时仍无 transition

  **单次标准修复**（按优先序）：

  1. 看 `D:\actions-runner\_diag\Runner_*.log` 最新一份末 30 行（在 ghrunner session 或拷出来看）：
     ```powershell

       Sort-Object LastWriteTime -Descending |
       Select-Object -First 1 |
       Get-Content -Tail 30
     ```
  2. 若日志显示网络错误 → 跑 `gh api user` 验证 token / 网络
  3. 若日志显示进程异常退出 → 在 ghrunner session 重启 `D:\actions-runner\run.cmd`
  4. 其他情况 → 记 backlog

  → 重跑 Step 1。仍 FAIL → **F.2 FAIL at C3**，记证据进 Task 3.4 FAIL 出口。
