P2 · CODEGEN SW Toolbox → 标准件几何

让购买件/标准件用真实 SolidWorks 模型，而不是简化的 CadQuery 形状
SW Toolbox *.sldprt 经 COM 导出为 STEP
适配器 adapters/parts/sw_toolbox_adapter.py；多尺寸件（一个 sldprt 出多规格）由 sw_config_broker（ShowConfiguration2）逐配置展开。
批量预热进共享 STEP 缓存
cad_pipeline.py sw-warmup 把 Toolbox 件批量转 STEP 进 ~/.cad-spec-gen/step_cache/（GB 330 件已缓存约 328/330），新项目零手工 YAML 也能命中真实模型。
产物 step_cache/ 真实 STEP · sw-warmup-smoke.log
parts_resolver 按优先级选模型
用户/项目 STEP → 共享 STEP 缓存 → ★ SW Toolbox ★ → bd_warehouse → PartCAD → CadQuery 兜底（jinja_primitive，永远最后）。命中即返回。
产物 std_*.py（真实 SW 几何）· geometry_report.json（A–E 质量分级，标出仍是 C/D/E 兜底的件）· resolve_report（路由结果，可在 sw_report.html 看）
诊断 / 只读计划
cad_pipeline.py sw-inspect（--deep 启 COM 查环境/索引/材质/产物）· sw-export-plan（只读生成 Toolbox STEP 导出候选计划）· sw-toolbox-e2e（真实 Toolbox → STEP → codegen 消费端到端验收）。

P4 · RENDER → P5 · ENHANCE SW 外观库 → PBR 材质

让 Blender 渲染和 AI 增强都用 SolidWorks 的外观/纹理，不是凭空捏的材质
SW 外观库纹理图片 → 环境变量
cmd_render 把 SwInfo.textures_dir 写进 SW_TEXTURES_DIR 环境变量（也接受 CAD_SPEC_GEN_TEXTURE_DIR）。
render_3d.py 解析贴图路径
_resolve_texture_path()：绝对路径直用，相对路径靠上面的 env 找；miss 只 log.warning + 退回标量颜色，不抛。
create_pbr_material 挂 4 类 PBR 贴图
base color / roughness / normal / metallic 四类贴图节点接进 Blender 材质；运行时还能用 sw_texture_backfill + load_runtime_materials_override（env JSON）回填 SW 外观。
产物 Blender Cycles 渲染的多视角 PNG（材质带 SW 外观）
统一 MATERIAL_PRESETS：一份外观喂两端
同一份 MATERIAL_PRESETS 既驱动 Blender PBR，又生成 AI enhance 的 prompt（含视角感知的 Fresnel/高光）→ SW 外观一路传到照片级 JPG，多视角一致。
产物 *_enhanced.png（照片级，材质继承 SW 外观）
⊕ 零配置「SW 装即用」检测 / 预检层（横跨各阶段）
sw_detectwinreg 探测 SW 安装路径、版本、edition（本机 → premium/professional）
sw_preflight/体检 + 一键修：Toolbox 索引、材质矩阵 M、缓存、用户提供件、HTML 报告
cad_pipeline.py env-check校验 Python / CadQuery / Blender / SW 等环境是否就绪
sw-inspect / sw_config_brokerCOM 深度诊断 · 多规格件配置广播 · 三层缓存 + prewarm
注：SW 是可选增强源而非硬依赖——没装 SW 时 resolver 自动退到 STEP 缓存 / bd_warehouse / PartCAD / CadQuery 兜底，材质退回标量颜色，管线照常跑完，只是部分件几何质量降到 C/D/E、材质少了真实贴图。所有 SW 接入点都有 env kill switch + conftest autouse 默认禁用（防 modal 卡 CI）。

06

AI 增强后端 · 4 选 1

自动探测：FAL_KEY → fal · ComfyUI 在跑 → comfyui · Gemini 配置在 → gemini · 都没有 → engineering
后端成本运行位置几何锁定方式适用场景
gemini~$0.02 / 图云软锁（prompt + 源图首位 + V1 锚 + 全分辨率）性价比首选；提案/答辩出图，质感好、成本低
fal~$0.20 / 图云（fal.ai Flux）硬锁（ControlNet 深度 + 边缘）对几何零容忍、要求严格贴合渲染图的场景
comfyui免费本地 GPU硬锁（本地 ControlNet 深度 + 边缘）有本地 GPU、要免费 + 硬锁、批量出图
engineering免费本地（无 AI）完美（Blender PBR 直接渲成 JPG）不要 AI、要 100% 几何保真的工程预览；任何环境兜底
降级链：fal → gemini → engineering（中途降级后整批锁定该后端以保一致性）· 增强永远跳过 *_enhanced.* 防重复处理 · 永久切换：pipeline_config.json > enhance.backend
