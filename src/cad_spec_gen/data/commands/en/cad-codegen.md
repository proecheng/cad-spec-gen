# /cad-codegen — Generate CadQuery Scaffold Code from CAD_SPEC.md

User input: $ARGUMENTS

## Instructions

Auto-generate CadQuery Python scripts from CAD_SPEC.md structured data via Jinja2 templates.

### Prerequisites

- Target subsystem directory must contain `CAD_SPEC.md` (generated by `/cad-spec`)
- Jinja2 must be installed (`pip install Jinja2`)

### Routing Rules

1. **No arguments** → Show usage:
   ```
   Usage: /cad-codegen <subsystem> [--force]

   Examples:
     /cad-codegen end_effector            # scaffold mode (won't overwrite existing files)
     /cad-codegen end_effector --force    # force overwrite all generated files
     /cad-codegen charging_dock           # subsystem name
   ```

2. **`<subsystem>`** → Execute code generation:
   ```bash
   python cad_pipeline.py codegen --subsystem <subsystem>
   ```

3. **`<subsystem> --force`** → Force overwrite mode:
   ```bash
   python cad_pipeline.py codegen --subsystem <subsystem> --force
   ```

### Generated Content

Code generation runs in 5 steps using generators in `codegen/` and Jinja2 templates in `templates/`:

| Step | Generator | Template | Input (CAD_SPEC) | Output |
|------|-----------|----------|-------------------|--------|
| 1 | `gen_params.py` | `params.py.j2` | §1 Global params table | `params.py` — dimension constants |
| 2 | `gen_build.py` | `build_all.py.j2` | §5 BOM tree | `build_all.py` — STEP/STD/DXF build table |
| 3 | `gen_parts.py` | `part_module.py.j2` | §5 BOM (custom leaf parts) | `station_*.py` — part scaffolds |
| 4 | `gen_assembly.py` | `assembly.py.j2` | §4 connections + §5 BOM + §6 pose | `assembly.py` — assembly structure (incl. std parts) |
| 5 | `gen_std_parts.py` | — | §5 BOM (purchased parts) | `std_*.py` — simplified standard part geometry |

### Standard Parts Auto-Generation (Step 5)

`gen_std_parts.py` extracts all `外购` (purchased) parts from BOM, auto-classifies by name/model, and generates simplified CadQuery geometry:

| Category | Simplified Geometry | Example Models |
|----------|-------------------|----------------|
| motor | cylinder + shaft | Maxon ECX SPEED 22L |
| reducer | wider cylinder | Maxon GP22C |
| spring | annular disc | DIN 2093 A6 |
| bearing | inner/outer rings | MR105ZZ, 688ZZ |
| sensor | cylinder/box | ATI Nano17, I300-UHF |
| pump | box + port stubs | gear pump |
| connector | small cylinder | LEMO FGG.0B |
| seal | torus | FKM O-ring |
| tank | cylinder | stainless steel tank |

- Skips `fastener` (too small) and `cable` (flexible)
- Dimensions from: `cad_spec_defaults.py` → `STD_PART_DIMENSIONS` lookup table
- Output naming: `std_ee_001_05.py` (`std_` prefix + part number suffix)
- Scaffold mode never overwrites existing files

### Scaffold vs Force Mode

- **scaffold** (default): Only generates files that don't exist; engineer-modified files are never overwritten
- **force**: Regenerate and overwrite all; suitable for first-time generation or after major CAD_SPEC changes

### Post-Generation Summary

After execution, report to user:
- Each generator's result (success/skipped/failed)
- List of generated or updated files
- Next step suggestions: `/mechdesign <subsystem>` to refine geometry, or `python cad_pipeline.py build` to build directly

### Relationship with Other Commands

```
/cad-spec → CAD_SPEC.md        (Phase 1: Normalization)
/cad-codegen → *.py scaffolds   (Phase 2: Code Generation)  ← You are here
/mechdesign → manual geometry    (Manual stage)
cad_pipeline.py build            (Phase 3: Build)
cad_pipeline.py render           (Phase 4: Render)
cad_pipeline.py full             (One-shot full pipeline)
```
