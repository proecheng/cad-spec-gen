# /cad-codegen — Generate CadQuery Scaffold Code from CAD_SPEC.md

User input: $ARGUMENTS

## Instructions

Auto-generate CadQuery Python scripts from the structured data in CAD_SPEC.md via Jinja2 templates.

### Prerequisites

- `CAD_SPEC.md` must already exist in the target subsystem directory (generated by `/cad-spec`)
- Jinja2 must be installed (`pip install Jinja2`)

### Routing Rules

1. **No arguments** → Show usage:
   ```
   Usage: /cad-codegen <subsystem> [--force]

   Examples:
     /cad-codegen end_effector            # scaffold mode (does not overwrite existing files)
     /cad-codegen end_effector --force    # force overwrite all generated files
     /cad-codegen charging_dock           # subsystem name or alias
   ```

2. **`<subsystem name>`** → Run code generation:
   ```bash
   python cad_pipeline.py codegen --subsystem <subsystem>
   ```

3. **`<subsystem name> --force`** → Force overwrite mode:
   ```bash
   python cad_pipeline.py codegen --subsystem <subsystem> --force
   ```

### Post-generation Auto-check (Gate 2)

`gen_parts.py` immediately scans all newly generated files for `TODO:` markers:

- **Unfilled TODOs found** → Print WARNING list (filename + line number + content), exit with **exit code 2**
- **All TODOs filled** → Print `All coordinate system blocks filled. Ready for build.`

Each generated `<part>.py` file header contains a mandatory coordinate system declaration block that **must** be filled before proceeding to Phase 3:

```python
# ┌─ COORDINATE SYSTEM (MUST fill before coding geometry) ──────────────────┐
# Local origin : <e.g. bottom-left corner of mounting face>
# Principal axis: <e.g. extrude along +Z (axial), body height = PARAM_H>
# Assembly orient: <e.g. rotate X+90deg → axis becomes +Y (radial)>
# Design doc ref : <e.g. §4.1.2 L176 — "tank axis collinear with cantilever (radial)">
# └──────────────────────────────────────────────────────────────────────────┘
```

> Three gates: Gate 1 (DESIGN_REVIEW CRITICAL) → Gate 2 (TODO scan) → Gate 3 (orientation_check.py)

Code generation runs in 5 steps using generators in `codegen/` and Jinja2 templates in `templates/`:

| Step | Generator | Template | Input (CAD_SPEC) | Output |
|------|-----------|----------|------------------|--------|
| 1 | `gen_params.py` | `params.py.j2` | §1 Global params table | `params.py` — dimension constants |
| 2 | `gen_build.py` | `build_all.py.j2` | §5 BOM tree | `build_all.py` — STEP/STL/DXF build table |
| 3 | `gen_parts.py` | `part_module.py.j2` | §5 BOM (fabricated leaf parts) | `station_*.py` — part scaffolds |
| 4 | `gen_assembly.py` | `assembly.py.j2` | §4 connections + §5 BOM + §6 poses | `assembly.py` — assembly structure (incl. standard parts) |
| 5 | `gen_std_parts.py` | — | §5 BOM (purchased parts) | `std_*.py` — simplified standard part geometry |

### Standard Part Auto-generation (Step 5)

`gen_std_parts.py` extracts all `purchased` parts from the BOM, auto-classifies them by name/model, then generates simplified CadQuery geometry:

| Category | Simplified Geometry | Example Models |
|----------|--------------------|-----------------|
| motor | Cylinder + shaft | Maxon ECX SPEED 22L |
| reducer | Cylinder (wider) | Maxon GP22C |
| spring | Annular disc | DIN 2093 A6 |
| bearing | Inner/outer rings | MR105ZZ, 688ZZ |
| sensor | Cylinder/box | ATI Nano17, I300-UHF |
| pump | Box + port | Gear pump |
| connector | Small cylinder | LEMO FGG.0B |
| seal | Torus | FKM O-ring |
| tank | Cylinder | Stainless tank |

- Skips `fastener` (too small) and `cable` (flexible body)
- Dimensions sourced from: `cad_spec_defaults.py` → `STD_PART_DIMENSIONS` lookup table
- Output naming: `std_ee_001_05.py` (`std_` prefix + part number suffix)
- In scaffold mode, existing files are not overwritten

### scaffold vs force Mode

- **scaffold**: Only generates files that don't exist; preserves files that engineers have manually edited
- **force** (default for `gen_params.py`): Regenerates and overwrites everything; use for initial generation or full reset after major CAD_SPEC changes

> **v2.0 change**: `gen_params.py` default mode changed from `scaffold` to `force` (always fully regenerates `params.py` on codegen). `gen_parts.py` and `gen_std_parts.py` added `--mode force` option. Pipeline `codegen --force` passes `--mode force` to all generators.

### Post-generation Summary

After completion, report to user:
- Execution result of each generator (success / skipped / failed)
- List of generated or updated files
- Recommended next step: `/mechdesign <subsystem>` to manually refine geometry details, or `python cad_pipeline.py build` to build directly

### Relationship to Other Commands

```
/cad-spec → CAD_SPEC.md       (Phase 1: specification)
/cad-codegen → *.py scaffold  (Phase 2: code generation)  ← you are here
/mechdesign → manual geometry  (manual stage)
cad_pipeline.py build          (Phase 3: build)
cad_pipeline.py render         (Phase 4: render)
cad_pipeline.py full           (one-shot full pipeline)
```
